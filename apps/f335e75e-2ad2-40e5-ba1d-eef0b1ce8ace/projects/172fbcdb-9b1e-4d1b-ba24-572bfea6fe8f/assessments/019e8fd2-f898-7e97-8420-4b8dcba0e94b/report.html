Core Business Workflows

The application supports currency conversion operations for banking scenarios by validating requested currency pairs, retrieving active exchange rates, and returning deterministic conversion outputs.

Service-to-Domain Mapping

ServiceDomain ContextOwned EntitiesExternal Dependencies
ZavaCurrencyServiceFX Quote and ConversionCurrency Pair, Exchange Rate, Currency Rate ResponseSQL Server database

Primary Workflows

Workflow 1: Retrieve Latest Exchange Rate

  1. Client invokes GetRate with source and target currency codes.
  2. Service normalizes and validates the currency inputs (3-letter format).
  3. Service queries active pair/rate rows ordered by most recent effective date.
  4. Service returns a rate payload or a fault when no active rate exists.

Workflow 2: Convert Amount

  1. Client invokes ConvertAmount with amount and currency pair.
  2. Service validates amount is non-negative.
  3. Service reuses rate retrieval workflow to resolve the latest rate.
  4. Service computes and rounds converted amount to four decimal places.
  5. Service returns converted value or a validation fault.

Cross-Service Data Flows

No cross-service aggregation flow is present in this repository. All business decisions execute within one service boundary, with direct reads from SQL Server and immediate response construction.

Business Workflow Sequence

sequenceDiagram participant Client participant CurrencyService as "Currency Service" participant DB as "SQL Server" Client->>CurrencyService: Request conversion (amount, from, to) CurrencyService->>CurrencyService: Validate amount and currency format alt Validation passed CurrencyService->>DB: Retrieve latest active rate for pair DB-->>CurrencyService: Pair and rate data CurrencyService->>CurrencyService: Compute rounded converted amount CurrencyService-->>Client: Conversion result else Validation or lookup failed Note over CurrencyService: Return business fault response CurrencyService-->>Client: Fault message end

Business Rules & Decision Logic

  • Amount must be non-negative before conversion can proceed.
  • Currency codes must be exactly three letters and are normalized to uppercase.
  • Only active currency pairs and non-expired rates are eligible for conversion.
  • Latest effective exchange rate is selected for deterministic conversion behavior.
  • Missing rate data produces an explicit fault instead of fallback estimation.

Version & Compatibility Risks

The project targets .NET Framework 4.8 and uses classic WCF server APIs (System.ServiceModel) that are not available as-is on modern .NET; this is a primary migration risk. Package versions are framework-bound references rather than NuGet-managed versions, reducing upgrade flexibility.

Notable Observations

  • No external NuGet dependencies are declared in packages.config.
  • WCF server hosting and System.Web couple the service to legacy ASP.NET hosting stacks.
  • Data access is direct SQL without ORM abstractions, which simplifies dependency count but increases migration effort.
  • No dedicated observability/logging library is declared.
